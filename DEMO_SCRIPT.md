## 슬라이드 2: 문제 정의 (0:30 ~ 1:30)

현재 많은 기업에서 렌탈이나 마케팅 예산을 배분할 때 경험과 직감에 의존합니다.

동일 비율로 배분하는 경우가 많죠.

그리고 결정적으로 실시간 데이터 기반 의사결정 시스템이 없습니다.
매달 엑셀로 수작업 분석하고, 한 달 뒤에야 결과를 확인합니다.

MoveSignal AI는 이 문제를 해결합니다.

MoveSignal AI는 세 가지 축으로 구성됩니다.

## 슬라이드 5: Feature Mart (3:30 ~ 4:30)

Feature Mart 설계를 보시면, 5개의 스테이징 테이블을 만들었습니다.

STG_POP은 유동인구 - 거주 인구, 직장 인구, 방문 인구를 구분합니다.
STG_CARD는 카드 소비 - 외식, 카페, 의류, 문화 등 8개 카테고리를 포함합니다.
STG_ASSET은 자산, STG_PRICE는 부동산 시세,
STG_MOVE는 전입/전출/순이동 데이터입니다.

이걸 모두 합쳐서 FEATURE_MART_FINAL -
3개 구 곱하기 60개월, 2021년 1월부터 2025년 12월까지의
통합 테이블을 만들었습니다.

## 슬라이드 9: 왜 중구가 67%인가 (8:00 ~ 8:45)

가장 많이 받는 질문이 "왜 중구가 67%나 되나요?" 입니다.

답은 간단합니다.

명동, 을지로, 충무로 - 서울 최대 상업 밀집 지역이고,
실제 카드 소비 절대 규모가 서초구와 영등포구의 4~5배입니다.

이건 모델이 만들어낸 숫자가 아니라
실제 SPH 카드 매출 데이터가 보여주는 팩트입니다.

그리고 만약 비율을 조절하고 싶다면
What-if 시뮬레이션으로 자유롭게 바꿔볼 수 있습니다.
AI에게 "영등포구에 예산을 늘려야 할까?" 물어볼 수도 있고요.

## 예상 Q&A 대비

**Q: 왜 3개 구만 했나요?**
해커톤 기간 내 검증을 위해 서울의 대표적 3개 상권을 선택했습니다.
Feature Mart 구조가 구 단위로 설계되어 있어서
25개 구로 확장은 SQL 한 줄 수정으로 가능합니다.

**Q: 중구 67%가 너무 치중된 것 아닌가요?**
이건 ML 모델이 만들어낸 숫자가 아니라 실제 카드 소비 데이터의 반영입니다.
명동/을지로 상권의 절대 소비 규모가 그만큼 크다는 것을 보여줍니다.
What-if 시뮬레이션 기능으로 원하는 비율로 조절해서 시뮬레이션할 수 있습니다.

**Q: Cortex LLM은 환각(Hallucination) 문제가 없나요?**
Feature Mart의 실제 데이터를 프롬프트에 컨텍스트로 주입하고 있어서,
순수 LLM 대비 환각이 크게 줄어듭니다.
또한 Fallback 로직으로 Cortex 실패 시 정적 데이터를 표시합니다.

**Q: 공공 활용이라고 했는데 실제로 가능한가요?**
같은 Feature Mart에서 구별 유동인구, 카드소비, 인구이동 데이터가 이미 구축되어 있습니다.
구청이 상권 활성화 예산을 배분할 때 필요한 데이터가 정확히 이겁니다.
Streamlit 앱에서 "행정 모드" 탭 하나만 추가하면 바로 사용 가능합니다.

**Q: 비용이 $80인데 ROI가 실제로 그만큼 나오나요?**
기존에 분석팀이 매월 Excel로 3~5일간 수작업하던 분석을 실시간으로 대체합니다.
인건비 절감만 계산해도 월 $80은 충분히 정당화됩니다.
